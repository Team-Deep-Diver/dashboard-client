# **프로젝트 소개**

**사용자가 작성한 카드들을 한 눈에 시각적으로 볼 수 있는 대시보드입니다.**

사용자는 카드를 생성하여 메모, 사진, 해야할 일들을 입력할 수 있습니다.
생성된 카드는 수정하고 삭제할 수 있으며 카드들은 원하는 곳에 배치할 수 있습니다.

# 💻 **실행 방법**

### **_Front_**

클라이언트 레포지토리를 클론받습니다.

```
git clone https://github.com/Team-Deep-Diver/dashboard-client.git
```

프로젝트 폴더 root 위치에 .env 파일을 만들고 다음과 같이 환경변수를 설정합니다.

```
REACT_APP_SERVER_REQUEST_HOST
REACT_APP_SOCKET_SERVER_HOST
```

client 프로젝트 폴더 터미널에서 아래 명령어를 실행합니다.

```
npm install
npm start
```

### **_Back_**

서버 레포지토리를 클론받습니다.

```
git clone https://github.com/Team-Deep-Diver/dashboard-server.git
```

프로젝트 폴더 root 위치에 .env 파일을 만들고 다음과 같이 환경변수를 설정합니다.

```
PORT
SOCKET_CLIENT_HOST
MONGOOSE_URL
JWT_SECRET
SESSION_SECRET
```

server 프로젝트 폴더 터미널에서 아래 명령어를 실행합니다.

```
npm install
npm run dev
```

# 🙏 **소감**

### **_사공은혜_**

저는 이번 프로젝트 기간 동안 팀워크와 소통의 중요성을 배울 수 있었습니다. 혼자 과제를 할 때에는 당연하게 생각했던 포인트들이 모두 소통의 일부분이라는 것을 느끼게 되었습니다.

예를 들어 만들고자 하는 어플리케이션의 세부적인 userflow가 어떻게 되는지, 프런트와 백엔드 간 주고받는 데이터 형식 및 응답 코드는 어떠한지 등 가장 기본적이고 세부적인 부분부터 잡 잡아야 충돌 없이 개발할 수 있음을 배울 수 있었습니다.

저 역시 처음 팀 프로젝트를 하다 보니 미숙했지만, 이번 기회를 통해 팀플의 기본을 배울 수 있어 좋았습니다. 이번에 느끼고 배운 점들을 바탕으로 좀 더 성장한 프로젝트를 이어나갈 수 있도록 하겠습니다.

<br />

### **_이세영_**

이번 팀플은 켄님과 멘토님들이 항상 강조하시던 한 줄 한 줄의 의미, 왜 쓰이는 가에 대해 파고 들어야 하는 이유와 중요성을 깨닫게 된 시간이었습니다.

맡은 부분에 대해 질문을 받을 때 명확한 답변을 하지 못하고, 논리적으로 설명하지 못하던 상황들이 많았는데, 그 이유가 두루뭉술하게 이해하고 넘어가서 생긴 문제점이라고 생각합니다.

문제점을 깨닫고, 켄님과 멘토님들이 말씀하시던 부분에 대해 생각하며 파고든 결과, 저의 코드에 대해 논리적으로 설명할 수 되었고, 이것이 결과적으로 팀원분들과 조금 더 수월하게 소통하게 되는 바탕이 됐습니다.

<br />

### **_한아름_**

저는 이번 프로젝트를 진행하면서 소통의 중요성, 소통의 힘을 많이 깨달았습니다.

제가 fe 작업을 하는 도중에 be로 부터 받은 응답 데이터 수정이 필요한 상황이 생겼습니다. 이미 백엔드 작업이 완료된 상태였는데 이 상황을 설명드리는게 처음에는 어려웠습니다. 그럼에도 불구하고 백엔드 작업을 맡으신 분한테 응답 데이터 수정요청이 필요한 상황을 설명을 드렸습니다. 이러한 과정이 여러번 있었는데 이러한 일련의 과정을 거치면서 백엔드, 프론트엔드 각각의 입장에서 생각해보고 소통하는 연습을 하게 되어서 너무 좋았습니다.

또한 프로젝트를 진행 도중 문제를 하나하나 맞닦뜨릴때마다 걱정이 많아지고 감정적으로 불안해졌습니다. 그러나 팀원들과 같이 소통하면서 그런 걱정에서 내려와서 실제로 우리가 해결해야 하는 문제가 정확히 무엇인지 그 문제가 발생하는 원인이 무엇이고 어떤 부분을 해결해야하는지 객관적으로 현실을 받아들이는 연습이 되었습니다. 문제를 해결하려면 꽁꽁 싸매서 걱정하지 말고 팀원들과 적극적으로 소통해야 함을 배웠습니다.
